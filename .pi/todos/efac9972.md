Implemented in src/main.js + index.html.

Behavior:
- `state.mode` ('ai' | '2p'), default 'ai'. Press `1`/`2` between rallies to toggle.
- HUD shows current mode (`1P vs AI` / `2P LOCAL`) between the scores.
- Help line documents the new keys.
- `updateAI(dt)` called from `tick` when mode is 'ai':
  - Drifts toward y=0 when ball moves away or during serve.
  - Tracks `ball.y + aiAimError` when ball heads to AI side.
  - `AI_PADDLE_SPEED = 11` (vs player's 14) so the player can still win.
  - `AI_MAX_ERROR = 1.0`, re-rolled every `resetBall` so rallies vary.
  - `AI_CENTER_DEADZONE = 0.25` only applies during the centering drift; tracking mode runs every frame for smooth motion.
- Movement clamped via existing `clampPaddle`.

Follow-up fix:
- Removed the deadzone in tracking mode (it was causing step-stop stutter against a hovering ball). `Math.min(step, |dy|)` already prevents overshoot, so no jitter guard is needed there.
- Bumped AI speed 9.5 -> 11 so the AI no longer feels sluggish vs the player.
